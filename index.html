Introduction

Futurerestore is a tool that allows iOS users to upgrade, downgrade, or re-restore to an unsigned firmware utilizing SHSH blobs. This is a guide that will focus on helping upgrade, downgrade, or re-restore using Futurerestore.

This guide is based off of this one, except it has been made into a website for better formatting.

Keep in mind that anything said in this guide is information that should be taken with a grain of salt, as the guide is targeted to users who want to use Futurerestore to upgrade to iOS 12.2 (as of August 2019), but is not limited to those users, as the methods used here can be used to go to another version, but with some modifications.

No one but you will be held resposible for any damage caused to your device when following this guide.

Background Info

Before you begin following this guide, there are few things to keep in mind:

  • As of August 2019, iOS 12.4's SEP and Baseband are fully compatible when downgrading/upgrading, or re-restoring to iOS 12.2. This means that you can use the latest SEP and Baseband method, instead of manually specifying your SEP and Baseband.
  • Make sure iTunes and its drivers are installed correctly. This is critical to Windows users, as you want to avoid having the Microsoft Store version of iTunes.
  • If you decide to use a backup after using Futurerestore, make sure that you have gone through the process of getting rid of your jailbreak before backing up your jailbroken device. Most recent jailbreaks have an option named "Restore RootFS" that can help with this.
  • This process does not work for 32-bit Windows users. Sorry!

Getting ready to Restore

Before the restore process happens, there are a few things you will need:

  • Futurerestore's latest released version.
  • Your SHSH blob file from TSSSaver. (If you're unsure as to which file you should get, obtain the file that is inside your noapnonce folder)
  • The IPSW file of the version you're going to upgrade, downgrade, or re-restore to. You can find beta firmwares here, otherwise, you can obtain officially released firmwares from ipsw.me.

Once you have all of these requirements downloaded, put them in a folder, which should be named futurerestore.

When you download your blob, it is recommended to rename it to blob.blob (for ease). Your IPSW should be renamed to something of your liking (such as ipsw.ipsw) for ease.

Finding your Nonce Generator

Finding and setting your Nonce Generator for your blob is a very critical step throughout this process, otherwise, the restoration process will not work. Here's how you can find your nonce generator for your blob:

  1. Download your blob from TSSSaver
  2. Open your blob file with a text editor
  3. (On Windows) Press CTrl +F, or (On macOS) Press Command + F, then, search for the word "generator"

Next to, or underneath the generator, you will see that there's a string of (around) 16 characters. Please save this key for later.

Setting up your Nonce Generator

Now that you have found your nonce generator, its time to set it. There are many nonce setters out there, but here's a list of the most well know:

  • Ph≈ìnixNonce for iOS 9.x.
  • NonceSet for iOS 10.x.
  • NonceSet1112 for iOS 11 - 11.1.2.
  • GeoNonceSetter12 for iOS 12 - 12.2.
  • System Info for iOS 8, 9, 10, and 11. (Available in Cydia, or any other package manager)

Alternatively, modern jailbreak tools, such as unc0ver, or Chimera, have an option to set a nonce generator on supported firmwares, so you can use them to set your nonce as well.

Here's how you can set-up your nonce:

  1. Open your nonce setter tool (or modern jailbreak tool)
  2. Find a text field with the words "generator", "nonce" or "set nonce"
  3. Within that region, type in your key that you saved from your blob
  4. Press the "Set Nonce" button (or hit "Jailbreak" if you're using a jailbreak tool)

Once you've set your nonce generator, you're ready to begin the restoration process.

Restoring

You are finally ready to restore your device to the desired unsigned firmware. Get ready because this a heck of a lot of things to do. Before restoring, there are things you should keep in mind:

  • Failing to disable "Find my iPhone/iPad/iPod" and/or having NO PLS Recovery installed before beginning the restoration process can cause the restore to not begin, or fail.
  • If your device is put into "Recovery Mode" during this process, do not panic. This is entirely normal. However, avoid any iTunes alerts by hitting the "Cancel" button on any iTunes alert that you recieve.

A good practice when using Futurerestore is to place your blob, iPSW of unsigned firmware, and Futurerestore into one folder. With that out of the way, there are many ways you can restore to an unsinged firmware. This guide will go through most (if not, all) ways of restoring.

Common Restore (Automatic SEP and Baseband)

This method is commonly used when the latest SEP and Baseband of the latest iOS version is compatible with the version you are trying to restore to. In order to downgrade/upgrade, or re-restore using this method:

  1. Open Command Prompt/Terminal
  2. Open your Futurerestore folder, then drag futurerestore into Command Prompt/Terminal, then type -t
  3. Drag your blob file (blob.blob)
  4. Type --latest-sep --latest-baseband, then drag your IPSW file

In the end, your result should look something like this (assuming your folder is on your Desktop, using Windows):

C:\Users\<youruser>\Desktop\futurerestore.exe -t <your blob (blob.blob)> --latest-sep --latest-baseband <your IPSW (ipsw.ipsw)>

For iPods and Wifi-Only iPads, you should not specify the latest baseband:

C:\Users\<youruser>\Desktop\futurerestore.exe -t <your blob (blob.blob)> --latest-sep --no-baseband <your IPSW (ipsw.ipsw)>

Once you are sure that you have the right command, hit enter and the restore should begin.

Mnaually specify SEP & Baseband

This method is used when the latest SEP and Baseband of the latest iOS version is not compatible with the version you are trying to downgrade/upgrade, or re-restore to.

When specifying a custom SEP and Baseband, you are essensially doing so solemly because that specific SEP and Baseband is compatible with the version you're trying to downgrade/upgrade, or re-restore to.

Finding the right BuildManifest, compatible SEP, & compatible Baseband

  1. Find the link of the iOS version that has a Baseband that is compatible with the version you are restoring to. (You can find these in ipsw.me)
  2. Head over to extract.me, and paste the link of the IPSW that contains the compatible SEP and Baseband
  3. Find the file of the compatible Baseband and download it. (Most commonly, the compatible Baseband contains the iOS version number, followed by a .bbfw file format)
  4. Find the file of the compatible SEP and download it. (Most commonly, the compatible SEP file will contain the boardconfig indetifier of your device (without the "AP"), followed by a .im4w file format)
  5. Find the BuildManifest.plist file and download it (This is often found near the bottom)

These downloaded files should be placed into your futurerestore folder before beginning the restoration process.

Restoring

  1. Open Command Prompt/Terminal
  2. Drag your futurerestore folder into your Terminal
  3. Drag your blob file, type -s, then drag your SEP file
  4. Type -b, then drag your Baseband file
  5. Type -p, then drag your BuildManifest file
  6. Type -m, then drag your BuildManifest file (again), then drag your IPSW

In the end, your result should look something like this (assumming your folder is in your desktop, using Windows):

C:\Users\<youruser>\Desktop\futurerestore.exe -t <your blob (blob.blob)> -s <your SEP> -b <your Baseband> -p <your BuildmManifest> -m <your BuildManifest> <your IPSW (ipsw.ipsw)>

For iPods and Wifi-Only iPads, you should not specify the baseband:

C:\Users\<youruser>\Desktop\futurerestore.exe -t <your blob (blob.blob)> -s <your SEP> --no-baseband -p <your BuildmManifest> -m <your BuildManifest> <your IPSW (ipsw.ipsw)>

Once you are sure that you have the right command, hit Enter and the restore should begin.

If you encouter any errors throughout the process, head to this page, which has a list of common errors you can encouter when using Futurerestore and how to solve them.
