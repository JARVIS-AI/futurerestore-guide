Common Errors

Of course, not everything goes butter smooth sometimes, and the process of futurerestoring has some roadblocks you might come across with.

This is documentation on the most common errors you will encounter when using Futurerestore and how to solve them. The documentation is based off of this, except it is much more simple to read and easier to follow.

Error Code -2: Could not put your device in recovery mode

Generally, this issue is caused becasue your device cannot be put into Recovery Mode. However, Meridian users most likely will encounter this error because Meridian's ldrestart is broken.

If you're on iOS 10, using the Meridian jailbreak, follow the step below. Otherwise, click here if you encouter the issue on other firmwares using another jailbreak.

  1. Download Meridian, and sideload it using Cydia Impactor
  2. Open the jailbreak app, set your nonce, and jailbreak
  3. Attempt to Futurerestore and wait until your terminal says Entering recovery mode
  4. When Futurerestore attempts to put your device intro recovery mode, hold Home + Power, until you see the Apple Logo. (For the iPhone 7, you use the Power + Vol buttons to put your device into recovery mode

After, Futurerestore should work just fine and the process should be successful, assuming you don't encounter further errors.

Error code -8 & -10 (Unable to send iBEC/Place device into restore mode)

If you encouter error code -8 (Unable to send iBEC), and/or error code -10 (Unable to place device into restore mode), here's what you should do:

  1. Head to Device manager under Control Panel (Windows)
  2. Locate "Apple Recovery (iBoot) USB Composite Device (at the bottom)
  3. Right click, choose "Uninstall device". You may see a tick box that allows you to uninstall the driver software as well, tick that

All the three Apple mobile device entries under USB devices will disappear. At this point, unplug the phone, re-plug it in, then attempt to restore again.

If the restore fails when your device screen turns green:

  1. Go back to device manager and repeat the driver uninstall process (described above)
  2. DO NOT disconnect your device
  3. Go back to Futurerestore once again and repeat the restoration process

The proccess should begin again and the device will restore sucessfully.

Small, but significantly important errors

These errors are pretty small, and they shouldn't take you long to solve. Here's a list of these small errors and how to solve them.

Error code -2: Could not put device in recovery mode

  • Assuming you are using the latest release of Futurerestore, update to MacOS Mojave.
  • This error can also be caused by NO PLS Recovery, a tweak that prevents your iOS device from being put into Recovery Mode by tools such as iTunes. If you have this tweak installed, it is recommended to uninstall it.
  • If you are using Windows, the error can occur by using a USB port that is NOT attached to your computer’s motherboard. Please switch to a port that is attached to the motherboard.

Error code -20

  • This error can occur when not specifying the right APNonce. Make sure you have set the right Nonce Generator for your blob and try again.

Error: unable to receive message from FDR……

  • This error is caused by attempting to upgrade, downgrade, or re-restore to an incompatible version. Make sure that you are attemptint to Futurerestore to a version you can actually Futurerestore to. You should also make sure that said version is compatible with the latest iOS Baseband and SEP, otherwise, specify the compatible SEP and Baseband.

If your device is stuck on Recovery Mode, use the command --exit-recovery to kick your device out of Recovery Mode. Alternatively, you can use iMazing to kick it out of Recovery Mode.

C:\Users\\Desktop\futurerestore.exe --exit-recovery

If your device is stuck on the "Update Completed" screen after using Futurerestore, use iMazing to bypass the "Update Completed" screen.
